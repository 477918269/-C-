HTTP存在安全隐患，而HTTPS的最终目的就是为了让我们消除安全隐患的。

一.我们先来谈谈HTTP存在的缺点：
1.通信使用明文，内容可能被监听。
使用明文通信就好比在裸奔，而互联网是一个公共资源，报文能够在网络中的任意一个地方被截获。
2.不验证对方的身份，因此可能遭遇伪装
有可能会接受无意义的信息
3.无法验证报文的完整性，报文可能已经被篡改
可能服务端接收到的信息是在中间传输过程中篡改过的。


二.HTTPS = HTTP + 加密 + 证书 + 安全性保护

加密:
一种方式是对管道进行加密，由于HTTP中没有加密机制，所以都是用HTTP和SSL或TSL组合使用，加密HTTP的通信内容

证书：
由值得信任的第三方机构颁发，用来证明服务端和客户端是实际存在的。

安全保护：
通过创造数字签名来保证通信的完整性。

三.HTTPS只是身披SSL外壳的HTTP
HTTPS并不是一种新的应用层协议，而是HTTP通信接口部分用SSL和TSL协议代替了。
HTTP是直接和TCP通信的，而HTTPS是通过HTTP协议先和SSL协议通信，再由SSL和TCP通信达成的。


四.通信原理：
1.对称加密（共享密钥加密）：
机制:双方先约定一种加密方式，并根据约定随机生成一个密钥，通信过程中发送方使用密钥加密，接收方使用密钥解密。
缺点：如果第一次交换约定方式时被第三方截获，第三方就知道了密钥的解密方式，导致加密就失去了意义。

2.非对称加密（公开密钥加密）
公开密钥使用一对非对称的钥匙，一把是公钥，一把锁是私钥。公钥所有人都能知道，密钥所有人都不能知道。
机制:发送密钥的一方使用对方公钥进行加密处理，对方接受后，再使用自己的私钥进行解密。
缺点：这种方法是很安全的，但是效率很低

3.HTTPS使用的混合加密机制
具体操作:
1.使用公开密钥加密方式安全的将稍后的共享密钥加密中要使用的密钥传到接收方
2.确保密钥安全的前提下，使用共享密钥加密的方式进行通信。
	
但是这种方式时存在缺陷的：
无法证明公共密钥本身就是货真价实的公共密钥，因为第三方入侵者可以在公共密钥传输的时候截取公共密钥，并替换公共密钥，导致后面共享密钥传输期间加密的内容都能被第三方所破解。

解决方法：
使用第三方机构颁发的证书。

具体操作：
1.服务器把自己的公开密钥登陆到第三方机构
2.第三方机构用自己的密钥向服务器的公开密钥添加数字签名和颁发公钥证书
3.服务器将添加了数字签名和证书的公共密钥发给客户端
4.客户端向第三方机构认证公钥证书的数字签名和证书，以确认真实性。
5.使用服务器的公开密钥对报文进行加密处理后发送
6.服务器用私有密钥对报文解密。
